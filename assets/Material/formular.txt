Vorgaben:

- x muss mit der API abgeglichen werden und entspricht 1000 - [Anzahl bereits bestellter Pakete]
- Ist x kleiner / gleich 0 wird das Formular nicht mehr angezeigt und die Meldung "Leider sind alle Probepakete vergriffen." angezeigt.
- Validierung mit Bootstrap Validator
- Alle mit * markierten Felder sind Pflichfelder und dürfen deshalb nicht leer sein.
- Das Feld [geschmacksrichtung*] besteht aus drei radio-buttons mit den Werten Blueberry, Lemongras und Strawberry. Eine zusätzliche Visualisierung der Geschmacksrichtungen durch eine Miniaturdose ist wünscheswert.
- Das Feld [email] muss als E-Mail Adresse validiert werden.
- Das Formular muss per POST an die API URL (Formular Post) geschickt werden.
- Wenn die API den Wert 'success' zurückgibt, wird das Formular ausgeblendet und die Meldung "Vielen Dank für Ihre Bestellung!" angezeigt.



API:

Formular Post
URL: http://energydrink.stage.mediadivision.ch/api/post.php?key=[USERKEY]
Beschreibung: Nimmt die Formulardaten entgegen und speichert die übergebenen Felder in einer Datenbank
Feldernamen: geschmacksrichtung, vorname, nachname, email, strasse, plz, ort, tel
Rückgabe: success | error

Daten Abrufen
URL: http://energydrink.stage.mediadivision.ch/api/count.php?key=[USERKEY]
Beschreibung: Gibt die Anzahl bereits bestellter Pakete zurück
Rückgabebeispiele: 0, 12, 400

Daten Reset
URL: http://energydrink.stage.mediadivision.ch/api/reset.php?key=[USERKEY]
Beschreibung: Löscht alle bereits eingegangenen Bestellungen aus der Datenbank und sollte nach dem Testen des fertigen Formulars durchgeführt werden
Rückgabe: success | error

[USERKEY] = Userkey aus userkey.txt
